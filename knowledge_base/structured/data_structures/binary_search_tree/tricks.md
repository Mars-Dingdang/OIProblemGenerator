- **维护 `size` 字段**：用于支持快速查询排名和第k小元素，每次插入、删除后必须更新。
- **重复元素处理**：使用 `count` 字段记录重复次数，避免多个相同键节点导致树不平衡。
- **中序遍历有序性**：BST 的中序遍历结果是升序序列，可用于验证结构正确性或转换为排序数组。
- **删除时的后继选择**：可选择左子树的最大值或右子树的最小值替代当前节点，通常选右子树最小值（successor）更直观。
- **旋转操作不改变中序遍历**：左旋和右旋是平衡调整的基础，且保持BST性质不变。
- **平衡性破坏的四种情况**：LL、RR、LR、RL 型失衡，通过单旋或双旋恢复，是AVL等平衡树的核心逻辑。
- **递归实现简洁**：插入、删除、查询均可用递归清晰表达，回溯时更新 `size` 等信息。
- **指针返回技巧**：`remove` 和 `insert` 返回新的根节点，便于父节点链接，简化内存管理。
